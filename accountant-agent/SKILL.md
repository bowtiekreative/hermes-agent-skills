## How to Build Financial Model in Sturppy

1. Navigate to **app.sturppy.com** and log in with saved credentials
2. Click **"New Project"** (top-right corner)
3. Template selection — two paths:
   - **Using a template**: Click "Entertainer or artist" → check subcategories ("Film/videography" + "Youtuber"). If a "Use this template" button appears, click it. If none appears, use the blank path instead.
   - **Blank template** (fallback): Click **"None of these templates work for me"** → then click **"Use this template"** on the blank/custom template. This creates an empty project you populate from scratch.
4. Fill in each section via the sidebar:
   - **Acquisition** — Marketing spend, subscriber growth targets, CAC assumptions
   - **Revenues** — All 7 revenue streams with 3-year projections (see references/financial-model.md)
   - **Expenses** — Per-episode budget ($19,050) scaled to production schedule (12→24 eps/year)
   - **Assumptions** — Industry benchmarks (RPM rates, conversion rates, churn, inflation)
   - **Statements** — P&L, Cash Flow, Balance Sheet auto-generated from above inputs
   - **KPIs** — Key metrics dashboard (subscribers, views, burn rate, runway, revenue vs expenses)
   - **Valuation** — Investor-ready valuation section
5. Export / Share with investors

### Known Sturppy Pitfalls
- **Browser tool may be unstable** with Sturppy (502 errors common, especially on `browser_click`). If the browser crashes mid-session, retry the login and navigation flow. The session may persist for a brief window.
- **Template button visibility**: After selecting "Entertainer or artist" → subcategories, a "Get Started" or "Use this template" button may not appear clearly. The **"None of these templates work for me" → blank template** path is more reliable — it always shows a "Use this template" button.
- **Verify project creation**: After clicking "Create" on the blank template, check the breadcrumb or navigate back to the dashboard to confirm the project actually saved. Browser crashes can silently eat the create operation.
- **Revenue wizard flow**: When adding a new revenue stream, Sturppy opens an onboarding wizard that asks product type and pricing type before the line item is created. You must complete this wizard (click "Next") before moving to the next stream or tab.
- **Don't combine template + blank**: Using the template selector and then the "None of these templates work for me" link resets the subcategory selections. Pick one path: either use a pre-built template with subcategories, or start from scratch with blank. Don't try both.

## Skills Backup Workflow

When instructed to back up skills to GitHub:

1. **Collect skills**: Find all agent-created custom skills at `~/skills/` — `find ~/skills -name "SKILL.md" -maxdepth 3` shows what exists
2. **Stage**: `mkdir -p /tmp/skills-backup && cp -r ~/skills/<name> /tmp/skills-backup/` for each skill to back up
3. **Init repo**: `cd /tmp/skills-backup && git init && git branch -m main && git config user.email "user@email.com" && git config user.name "User Name"`
4. **Check auth**: Verify `gh` CLI is auth'd — `gh auth status` or use `$GITHUB` env var with `echo "$GITHUB" | gh auth login --with-token`
5. **Commit**: `git add -A && git commit -m "Initial backup: <comma-separated skill names>"`
6. **Create remote**: `gh repo create <owner>/hermes-agent-skills --public --description "Hermes Agent custom skills backup" --source . --remote origin --push`
7. **Verify**: Confirm the output URL — it's the canonical backup location
